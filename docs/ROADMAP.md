## 三、剩余课程规划（第11-20课）

### 第11课：全国广电CDN域名库建立
**目标**：不依赖已有URL，主动建立全国31省广电CDN域名清单

**方法**：
- 研究各省广电官网域名规律（不是搜索URL，是研究域名本身）
- 每个省的广电CDN通常有固定的域名格式，如：
  - 浙江：`*.cztv.com` / `*.cztvcloud.com`
  - 湖南：`*.hntv.tv`
  - 广东：`*.gdtv.cn` / `*.ngtv.cn`
  - 江苏：`*.jstv.com`
  - 北京：`*.brtv.cn`
- 建立文件：`src/data/cdn_domains.json`，结构：
  ```json
  {
    "浙江": {
      "domains": ["cztv.com", "cztvcloud.com"],
      "url_pattern": "/channels/lantian/{code}/{res}.m3u8",
      "known_codes": ["channel001", "channel002", ...]
    },
    "湖南": {...},
    ...
  }
  ```

**输出**：覆盖全国的CDN域名库，后续课程的基础数据

### 第12课：全国广电CDN频道代码探测器
**目标**：对每个省的CDN系统，探测所有可能的频道代码，发现未收录频道

**方法**：
- 针对每个省的URL规律，构造探测列表
- 并发探测（100线程），记录存活的频道代码
- 对新发现的频道，尝试识别频道名（从返回内容的title或已知映射推断）

**输出**：`src/data/cdn_channels.json`，各省频道代码完整映射表

---

### 第13课：酒店IPTV系统全国扩展
**目标**：从13个已知酒店系统IP出发，扩展到全国更多酒店系统

**现状**：已发现13个酒店IP，合计742个存活频道
**问题**：这13个IP是从现有数据里提取的，数量太少

**方法**：
- 研究酒店IPTV系统的URL规律：`/hls/[N]/index.m3u8`、`/tsfile/live/[N].m3u8`
- 用FOFA手动导出中国境内开放了这类接口的公网IP列表（备用渠道B）
- 对每个IP，并发探测hls/1到hls/200的存活频道
- 建立频道号→频道名的映射（通过对比多个系统的相同频道号推断）

**输出**：`output/hotel_channels.json`，酒店IPTV频道库

---

### 第14课：多渠道合并与智能去重
**目标**：把GitHub源、广电CDN、酒店IPTV三路数据合并，智能去重

**合并规则**：
- 相同频道名的URL，按Tier排序（Tier1优先）
- 相同URL在不同渠道出现，只保留一条
- 每频道最多保留：Tier1最多10条，Tier2最多5条，Tier3/4最多3条
- 广电CDN的URL标记为`source: cdn`，GitHub源标记为`source: github`

**输出**：`output/channels_master.json`，主数据库

---

### 第15课：定时自动更新调度器
**目标**：实现每6小时自动运行一次完整的数据更新流程

**方法**：
- 新建 `src/scheduler.py`，作为总入口
- 流程：抓取→解析→标准化→测速→分层→CDN探测→合并→输出
- 每6小时触发一次（Windows任务计划程序）
- 每次运行后输出运行报告：`output/run_report_{timestamp}.json`
- 保留最近7次运行报告，自动清理旧的

**Windows任务计划程序配置命令**（一键配置）：
```
schtasks /create /tn "IPTV-Engine" /tr "python E:\VSCODE\iptv-engine\src\scheduler.py" /sc hourly /mo 6 /st 00:00
```

### 第18课：GitHub仓库自动发现升级
**目标**：让discover.py每次运行都能自动发现新的高质量仓库

**当前问题**：发现的新仓库太少（只有6个），且KNOWN_REPOS黑名单要手动维护
**解法**：
- 把discover.py的结果写入 `src/data/known_repos.json`，自动累积
- 每次运行自动从该文件读取已知仓库列表，不再硬编码
- 对新发现的仓库，自动验证其m3u文件有效性后才加入SOURCE_LIST

### 第19课：FOFA备用渠道接入（B渠道）
**目标**：在FOFA付费账号可用的情况下，自动搜索全国公网udpxy服务

**FOFA搜索语法**：
```
body="udpxy" && country="CN" && status_code="200"
```
**预期结果**：几百个暴露在公网的udpxy服务，每个服务背后是一套运营商IPTV频道

**实现方式**：
- 如果有FOFA付费账号：调用API自动化
- 如果只有免费账号：在FOFA网页手动搜索，导出CSV，脚本读取CSV批量探测
- 对每个udpxy服务，探测其频道列表（通常有/status页面列出所有组播地址）

## 六、关键决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 技术栈 | Python（组件1）+ Node.js（组件2） | Python擅长数据处理，Node.js擅长API |
| 镜像策略 | 直连优先，镜像已废弃 | 本机直连GitHub最快（1.3s） |
| 并发策略 | 分批4并发，90秒超时 | 避免连接池打爆，兼顾大文件 |
| URL排序 | Tier1优先，同Tier按延迟排 | 官方CDN最稳定 |
| 每频道URL上限 | 20条（测速阶段） | 防止测速时间过长 |
| FOFA | 保留为备用渠道B | 免费账号API限制太多 |
| 更新频率 | 每6小时一次 | 平衡新鲜度和服务器负担 |
| 目标用户 | 大陆+海外华人（C类） | 需要全覆盖+稳定性 |
